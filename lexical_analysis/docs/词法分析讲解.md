# 词法分析（Lexical Analysis）讲解

## 一、什么是词法分析

词法分析是编译过程的**第一个阶段**。它的任务是：**从左到右逐个字符地扫描源程序字符流，按照词法规则识别出具有独立意义的最小语法单位——单词（Token）**，并输出对应的词法单元序列，作为语法分析器的输入。

可以把它理解为：把一长串原始字符 `"int x = 10 + 20;"` 切分成"有意义的词块"的过程。

```
源程序字符流  →  [词法分析器]  →  Token 序列
"int x=10;"     →  [(INT,"int"), (ID,"x"), (ASSIGN,"="), (NUM,10), (PLUS,"+"), (NUM,20), (SEMI,";")]
```

## 二、词法分析的作用

### 1. 简化后续阶段的工作
- **语法分析**只需处理 Token 流，无需关心空格、注释、换行等无关字符。
- 把"识别字符组合"这种细粒度工作与"识别语法结构"这种粗粒度工作解耦，符合分阶段编译的设计原则。

### 2. 提升编译效率
- 词法分析模式一般用**正则文法**描述，结构简单，可以用**自动机**（DFA）高效执行，时间复杂度为 O(n)。
- 若让语法分析器直接处理字符流，会导致语法分析状态空间爆炸。

### 3. 屏蔽输入差异
- 统一管理关键字、标识符、常量、运算符、界符等 token。
- 屏蔽空格、注释、换行、制表符等"无意义"字符。
- 统一错误提示位置（行号、列号）。

### 4. 提供符号表的初始信息
- 识别出的标识符（ID）送入符号表，供后续阶段查询。
- 常量字面量（NUM、STR）记录其字面值与类型。

## 三、词法分析器的输入与输出

### 输入
- 源程序文件（.c / .go / .java / 自定义 DSL 等），按字符流读取。

### 输出
- 一串 **Token**，每个 Token 形如 `(种别码, 属性值)`：
  - 种别码（Category）：表示这个 token 属于哪一类（关键字、标识符、整数、运算符……）。
  - 属性值（Value）：该 token 的具体字面值或符号表入口。

常见 Token 类别示例：

| 类别 | 示例 | 种别码示例 |
| --- | --- | --- |
| 关键字 | `if`, `int`, `for` | KW_IF, KW_INT |
| 标识符 | `x`, `sum`, `_count` | ID |
| 整数常量 | `123`, `0`, `0x1F` | NUM |
| 实数常量 | `3.14`, `1.0e-5` | REAL |
| 字符串常量 | `"hello"` | STR |
| 运算符 | `+`, `-`, `*`, `/`, `==` | OP |
| 界符 | `;`, `,`, `(`, `)`, `{`, `}` | SEP |

## 四、词法分析的两大核心工具

### 1. 正则表达式（Regular Expression, RE）
用来**形式化描述**每种 token 的模式。

例如：
- 标识符：`letter (letter | digit)*`
- 整数：`digit+`
- 关键字：固定字符串集合

### 2. 有限自动机（Finite Automaton, FA）
用来**机械地执行**识别过程。

- **NFA（非确定有限自动机）**：从正则表达式直接构造，状态转移不确定，可能有 ε 边。
- **DFA（确定有限自动机）**：每个状态对每个输入字符只有**唯一**的下一状态，**无 ε 边**，执行效率高。

#### NFA → DFA 的转换（子集构造法）
将 NFA 的"状态集合"压缩为 DFA 的"一个状态"，从而把不确定性消除。

#### DFA 最小化
合并**等价状态**，得到状态数最少的 DFA，节省存储和匹配时间。

## 五、本项目中的 DFA 设计

本作业的核心是**手写若干 DFA** 来识别不同类别的 token，再组合成一个**词法分析器主控程序**（通常为 DFA 调度器：依次尝试每个 DFA，取最长的匹配）。

### 通用结构

```go
// 状态转移表：state × charClass → nextState
var transTable [states][charClasses]int

// 接受状态集合：标记哪些状态表示识别成功
var acceptStates = map[int]TokenType{...}
```

### 状态转移表驱动法

用一张二维表 `transTable[当前状态][字符类别] = 下一状态` 模拟 DFA 的运行：

```go
state := 0
for i := start; i < len(input); i++ {
    cls := classify(input[i])  // 把字符归类
    next := transTable[state][cls]
    if next == -1 {            // 没有转移，识别结束
        break
    }
    state = next
}
if acceptStates[state] {
    // 识别成功，得到一个 token
}
```

### 字符归类（classify）
为了把 DFA 状态数控制得小，常把字符先**聚类**：
- `letter`、 `digit`、 `+`、 `-`、 `*`、 `/`、 `(`、 `)`、 `;`、 `_`、其它
每个类别只占 DFA 表中的一列。

## 六、词法分析器的使用方法

### 1. 准备输入
准备一份待分析的源代码文件，例如 `input.txt`：

```c
int x = 10;
if (x > 0) { x = x - 1; }
```

### 2. 编写 DFA
针对每种 token 类别分别写一个 DFA，例如：
- 标识符 DFA
- 整数 DFA
- 浮点数 DFA
- 运算符 DFA
- 关键字：识别出标识符后**查表**判断是否是关键字

### 3. 主控流程（最长匹配原则）
```
循环：
  1. 跳过空白字符（空格、Tab、换行）
  2. 从当前位置尝试所有 DFA，记录能够到达接受状态的最长前缀
  3. 若存在匹配：
        输出对应的 token
        输入指针前进最长匹配的长度
     若无匹配：
        报告词法错误
        输入指针前进 1（错误恢复）
直到输入结束
```

### 4. 输出结果
以 `(种别, 值, 行号, 列号)` 的形式输出所有 token，例如：

```
(1, "int", 1, 1)     // 1 表示关键字种别码
(2, "x",   1, 5)     // 2 表示标识符
(3, "=",   1, 7)     // 3 表示赋值号
(4, 10,    1, 9)     // 4 表示整数常量
(5, ";",   1, 11)    // 5 表示分号
(6, "if",  2, 1)
...
```

### 5. 错误处理
遇到无法识别的字符时，应**报告错误并继续**（错误恢复），常见策略：
- 跳过该字符
- 跳过整行
- 插入/删除一个字符（仅在自动纠错时使用）

## 七、示例：从正则表达式到 DFA

以**带符号整数** `(+|-|ε)digit+` 为例：

### 正则表达式
```
(+ | - | ε) digit (digit)*
```

### NFA（简画）
```
  +  -  digit
→ ○ → ○ → (○) → ○ → (接受)
```
- 状态 0：起始
- 状态 1：已读到符号
- 状态 2：已读到至少一个数字（接受）
- 状态 3：数字循环（接受）

### DFA（化简后）
| 当前状态 | + | - | digit | other |
| --- | --- | --- | --- | --- |
| 0 | 1 | 1 | 2 | — |
| 1 | — | — | 2 | — |
| 2* | — | — | 2 | — |

`2` 为接受状态，`*` 表示可接受。

## 八、常见考点速查

1. **给出 token，写出正则表达式**
2. **给出正则表达式，画出 NFA**
3. **NFA → DFA**（子集构造法）
4. **DFA 最小化**（划分等价类）
5. **设计 DFA 并用代码实现**（状态转移表法 / 状态转移函数法）
6. **最长匹配原则**与 DFA 调度器

## 九、如何从多个 Token 构造一个"大 DFA"

之前我们讲了如何为单个 token 写 DFA；工业级实现通常是**把所有 token 合并成一个大 DFA**，由 lex/flex 等工具自动生成。下面给出完整流程与示例。

### 1. 完整 6 步流程

```
写每个 token 的正则
       │
       ▼
Thompson 构造：每个 token → NFA
       │
       ▼
合并 NFA：新加 S0 + ε 边到每个 NFA 的起点
       │
       ▼
子集构造法：NFA → DFA
       │
       ▼
划分等价类：DFA 最小化
       │
       ▼
接受态打 token 标签
       │
       ▼
Go 代码：状态表 + Run() + 最长匹配调度
```

### 2. 示例：3 个 token 合并

| Token | 正则 | 例子 |
| --- | --- | --- |
| 整数 NUM | `[0-9]+` | `123` |
| 标识符 ID | `[a-zA-Z]([a-zA-Z]\|[0-9])*` | `x`, `sum1` |
| 加号 PLUS | `+` | `+` |

#### Step 1：每个 token 构造 NFA（Thompson）

**NUM 的 NFA**
```
        0-9
→ q0 ────→ q1 ────→ (q2)
                ↺ 0-9
```

**ID 的 NFA**
```
        a-zA-Z
→ p0 ──────→ p1 ──────→ (p2)
                ↺ a-zA-Z|0-9
```

**PLUS 的 NFA**
```
→ r0 ──+──→ (r1)
```

#### Step 2：合并 NFA

加一个超级起始状态 S0，用 ε 边分别连到 3 个 NFA 的起点：

```
                ε             ε             ε
            ┌────────┐    ┌────────┐    ┌────────┐
   → S0 ──→│  NUM   │──→ │  ID    │──→ │  PLUS  │
            │ start  │    │ start  │    │ start  │
            └────────┘    └────────┘    └────────┘
                │             │              │
              q0→q1→(q2)   p0→p1→(p2)     r0→(r1)
```

#### Step 3：子集构造法（NFA → DFA）

**核心思想**：把 NFA 中"多个状态组成的集合"当成 DFA 的"一个状态"。

**算法**
```
worklist = [ε-closure({S0})]   // 初始 DFA 状态
while worklist 不空:
    取出一个 DFA 状态 S
    for 每个字符类 c:
        T = ε-closure(move(S, c))
        if T 不为空:
            把 T 当作新 DFA 状态
            S --c--> T
```

**手动演算**

初始 DFA 状态 **D0** = `ε-closure({S0})` = `{S0, q0, p0, r0}`

| 字符类 | 经过的 NFA 状态 | ε-closure = DFA 状态 | 是否接受 |
| --- | --- | --- | --- |
| `0-9` | `q0→q1` | `{q1, q2}` = **D1** | ✓（含 q2） |
| `a-zA-Z` | `p0→p1` | `{p1, p2}` = **D3** | ✓（含 p2） |
| `+` | `r0→r1` | `{r1}` = **D4** | ✓（含 r1） |
| 其它 | 空 | **D_dead**（死状态） | ✗ |

继续展开：
- **D1 = {q1, q2}**：读 `0-9` 回到 D1；读其它 → 死状态
- **D3 = {p1, p2}**：读 `0-9` 或 `a-zA-Z` 回到 D3；读其它 → 死状态
- **D4 = {r1}**：无出边 → 死状态

#### 得到的 DFA 转移表

| 状态 | 0-9 | a-zA-Z | + | 其它 |
| --- | --- | --- | --- | --- |
| **D0** (起始) | D1 | D3 | D4 | D_dead |
| **D1** (接受) | D1 | D_dead | D_dead | D_dead |
| **D3** (接受) | D3 | D3 | D_dead | D_dead |
| **D4** (接受) | D_dead | D_dead | D_dead | D_dead |
| D_dead | D_dead | D_dead | D_dead | D_dead |

#### Step 4：给接受状态打标签

| 接受状态 | 标签 | 含义 |
| --- | --- | --- |
| D1 | **NUM** | 整数 |
| D3 | **ID** | 标识符 |
| D4 | **PLUS** | 加号 |

> 真实情况常常是同一状态对应多种 token（如 D3 既可能是 `if` 也可能是 ID），需要**附加判定**（如查关键字表），通常在 emit token 时做。

#### Step 5：最小化（划分等价类）

**思路**
1. 把状态分为**接受态**和**非接受态**两类
2. 看每一对状态在所有输入下"目标类"是否相同，相同则等价
3. 反复划分，直到不动点

> 本例 D1/D3/D4 转移表各异，已经是最小化结果。

#### Step 6：用 Go 代码实现

```go
type DFA struct {
    trans   [numStates][numClasses]int   // -1 表示无转移
    accept  [numStates]int               // -1 非接受；其它 = 种别码
    start   int = 0
}

// 状态表驱动的 DFA 运行（带最长匹配）
func (d *DFA) Run(input string) (Token, int) {
    state := d.start
    lastAccept := -1
    lastAcceptPos := -1
    pos := 0
    for pos < len(input) {
        cls := classify(input[pos])
        next := d.trans[state][cls]
        if next == -1 { break }
        state = next
        pos++
        if d.accept[state] != -1 {
            lastAccept = d.accept[state]
            lastAcceptPos = pos
        }
    }
    if lastAccept == -1 { return Token{Type: ERR}, pos }
    return Token{Type: lastAccept, Val: input[:lastAcceptPos]}, lastAcceptPos
}
```

> `lastAcceptPos` 是**最长匹配**的关键：每到一个接受态就记下，最后一个就是最长匹配。

### 3. 关键点速记

1. **合并 NFA** = 新起点 + ε 边
2. **子集构造法** = "NFA 状态集合"作为 DFA 的一个状态
3. **接受态** = 闭包内含 NFA 接受态的 DFA 状态
4. **关键字 vs 标识符** 冲突 → emit 时**查关键字表**
5. **最长匹配** → 跑 DFA 时记下每次到达接受态的位置，取最后一个
6. **DFA 最小化** → 划分等价类，合并相同目标类的状态

## 十、参考代码骨架（Go）

```go
package main

// Token 类型
type TokenType int

const (
    T_EOF TokenType = iota
    T_KW
    T_ID
    T_NUM
    T_OP
    T_SEP
)

type Token struct {
    Type TokenType
    Val  string
    Line int
    Col  int
}

// DFA: 标识符与关键字识别
func scanID(input string, start int) (Token, int) {
    state := 0
    i := start
    for i < len(input) {
        ch := input[i]
        switch state {
        case 0:
            if isLetter(ch) {
                state = 1
                i++
            } else {
                return Token{}, start
            }
        case 1:
            if isLetter(ch) || isDigit(ch) {
                i++
            } else {
                state = 2
            }
        case 2:
            // 接受
            lexeme := input[start:i]
            if isKeyword(lexeme) {
                return Token{Type: T_KW, Val: lexeme}, i
            }
            return Token{Type: T_ID, Val: lexeme}, i
        }
    }
    return Token{}, start
}
```

> **总结**：词法分析的核心 = **正则表达式（描述）+ DFA（执行）+ 最长匹配调度（组合）**。本作业的 DFA 生成与运行，就是把这三步用代码实现出来。
